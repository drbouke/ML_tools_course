Production ML Validation — Customer Churn Model

Topics: Model validation, Confusion Matrix, Accuracy, Precision, Recall, F1

Scenario
You are an ML Engineer. Recently, your team deployed a customer churn prediction model to production.

A monitoring job captured a small batch of predictions and their true outcomes. You must validate the model behavior before trusting it.

Prediction = model output, Actual = true outcome.
1 = churn, 0 = stayed.
Positive class is fixed: Churn = 1

Step 1 — Label Each Row

For each row, choose the correct label. Please attach a screenshot of your labels after generating the report: TP, FP, TN, FN.

# Prediction Actual Your Classification Status
Rules (Positive = 1)
  • TP: Pred=1, Actual=1
  • FP: Pred=1, Actual=0
  • FN: Pred=0, Actual=1
  • TN: Pred=0, Actual=0
Why this matters
  • FP → wasted retention actions
  • FN → missed churn risk
  • Precision/Recall show different failure modes
